## Architecture

### Overview

The localization system uses:
- **Resource Files (.resx)**: Store translated strings for each component
- **IStringLocalizer<T>**: Dependency injection for accessing localized strings
- **UserPreferenceCultureProvider**: Custom provider that reads user's language preference from the database
- **Cookie-based fallback**: For non-authenticated users

### Designer.cs Files

Each base `.resx` file has a corresponding `.Designer.cs` file that provides strongly-typed access to resources. These are auto-generated by Visual Studio's ResXFileCodeGenerator tool.

Example structure:
```
GroupSettingsModal.resx           # Base English strings
GroupSettingsModal.Designer.cs    # Auto-generated strongly-typed class
GroupSettingsModal.ko.resx        # Korean translations
GroupSettingsModal.de.resx        # German translations
... (other languages)
```

## Configuration

### Program.cs Setup

Localization is configured in `Program.cs`:

```csharp
// Add localization services
builder.Services.AddLocalization(options => options.ResourcesPath = "Resources");

// Configure request localization
builder.Services.Configure<RequestLocalizationOptions>(options =>
{
    var supportedCultures = new[]
    {
        new CultureInfo("en"),
        new CultureInfo("ko"),
        new CultureInfo("de"),
        new CultureInfo("es"),
        new CultureInfo("fr"),
        new CultureInfo("it"),
        new CultureInfo("ja"),
        new CultureInfo("zh"),
        new CultureInfo("ar"),
        new CultureInfo("nl"),
        new CultureInfo("pl"),
        new CultureInfo("pt"),
        new CultureInfo("ru")
    };

    options.DefaultRequestCulture = new RequestCulture("en");
    options.SupportedCultures = supportedCultures;
    options.SupportedUICultures = supportedCultures;

    // Add custom user preference culture provider (highest priority)
    options.RequestCultureProviders.Insert(0, new UserPreferenceCultureProvider());
});
```

### UserPreferenceCultureProvider

The `UserPreferenceCultureProvider` class reads the user's language preference from the database:

```csharp
/// <summary>
/// Custom culture provider that reads the user's Culture preference from the database
/// and applies it to the request culture for localization.
/// </summary>
public class UserPreferenceCultureProvider : RequestCultureProvider
{
    public override async Task<ProviderCultureResult?> DetermineProviderCultureResult(HttpContext httpContext)
    {
        // Only apply for authenticated users
        if (httpContext.User?.Identity?.IsAuthenticated != true)
            return null;

        var userId = httpContext.User.FindFirst(ClaimTypes.NameIdentifier)?.Value;
        if (string.IsNullOrEmpty(userId))
            return null;

        var userPreferenceService = httpContext.RequestServices.GetService<UserPreferenceService>();
        var culture = await userPreferenceService.GetPreferenceAsync(userId, PreferenceType.Culture);

        if (!string.IsNullOrEmpty(culture) && IsValidCulture(culture))
            return new ProviderCultureResult(culture, culture);

        return null;
    }
}
```

## Using Localization in Components

### Razor Components

Inject `IStringLocalizer<T>` where T is the resource class:

```razor
@using Microsoft.Extensions.Localization
@inject IStringLocalizer<FreeSpeakWeb.Resources.Pages.Home> Localizer

<PageTitle>@Localizer["PageTitle"]</PageTitle>

<h1>@Localizer["Title"]</h1>

<button>@Localizer["CreatePostButton"]</button>
```

### With Parameters

Use format strings with placeholders:

```razor
@* In resx file: UploadHint = "Up to {0} images вЂў PNG, JPG, GIF вЂў Max {1}MB each" *@
<p>@string.Format(Localizer["UploadHint"], maxImages, maxFileSizeMB)</p>

@* Or: AddImages = "Add {0} Images" *@
<button>@string.Format(Localizer["AddImages"], selectedCount)</button>
```

### In Code-Behind

```csharp
@code {
    [Inject]
    private IStringLocalizer<FreeSpeakWeb.Resources.Pages.Notifications> Localizer { get; set; } = default!;

    private string GetLoadingMessage()
    {
        return Localizer["LoadingNotifications"];
    }
}
```

## Adding a New Language

### Step 1: Create Resource Files

For each existing `.resx` file, create a new language-specific version:

```
Example: Adding Hindi (hi)
- Copy Login.resx to Login.hi.resx
- Translate all <data> values
- Repeat for all resource files
```

### Step 2: Update Program.cs

Add the new culture to the supported cultures list:

```csharp
var supportedCultures = new[]
{
    // ... existing cultures
    new CultureInfo("hi"),  // Add Hindi
};
```

### Step 3: Add Language Option to UI

Update the language selector component to include the new language option.

## Adding New Localizable Strings

### Step 1: Add to Base Resource File

Open the base `.resx` file (e.g., `Home.resx`) and add a new entry:

```xml
<data name="NewFeatureTitle" xml:space="preserve">
    <value>New Feature</value>
</data>
```

### Step 2: Regenerate Designer.cs

The Designer.cs file is auto-generated. In Visual Studio:
1. Right-click the `.resx` file
2. Click "Run Custom Tool" or save the file

### Step 3: Add Translations

Add the same key to all language-specific files:

```xml
<!-- Home.ko.resx -->
<data name="NewFeatureTitle" xml:space="preserve">

```

## Best Practices

### Resource Key Naming

- Use PascalCase for key names
- Be descriptive but concise
- Group related keys with prefixes when appropriate

```xml
<!-- Good -->
<data name="PageTitle" xml:space="preserve">
<data name="SaveChanges" xml:space="preserve">
<data name="LoadingNotifications" xml:space="preserve">

<!-- Avoid -->
<data name="txt1" xml:space="preserve">
<data name="button_save" xml:space="preserve">
```

### Placeholders

Use numbered placeholders for dynamic content:

```xml
<data name="WelcomeMessage" xml:space="preserve">
    <value>Welcome, {0}!</value>
    <comment>{0} is the user's name</comment>
</data>

<data name="ItemCount" xml:space="preserve">
    <value>{0} of {1} items</value>
    <comment>{0} is current count, {1} is total count</comment>
</data>
```

### Comments

Add comments to explain context for translators:

```xml
<data name="Submit" xml:space="preserve">
    <value>Submit</value>
    <comment>Button text for form submission</comment>
</data>
```

### Avoid Concatenation

Don't concatenate strings - use placeholders instead:

```csharp
// Bad - word order varies by language
var message = Localizer["You have"] + " " + count + " " + Localizer["notifications"];

// Good
var message = string.Format(Localizer["NotificationCount"], count);
// Where NotificationCount = "You have {0} notifications"
```

## Troubleshooting

### Missing Translations

If a translation key is missing, the localizer returns the key name. Check:
1. The key exists in the language-specific `.resx` file
2. The key name matches exactly (case-sensitive)
3. The `.resx` file is included in the project

### Culture Not Applied

If the correct language isn't being applied:
1. Check the user's Culture preference in the database
2. Verify the culture is in the supported cultures list
3. Check the order of culture providers in `RequestLocalizationOptions`

### Designer.cs Not Generated

If strongly-typed access isn't working:
1. Right-click the `.resx` file в†’ Properties
2. Set "Custom Tool" to `PublicResXFileCodeGenerator`
3. Set "Custom Tool Namespace" to the appropriate namespace
4. Save and rebuild
